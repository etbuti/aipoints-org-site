AuditVisa Evidence Specification v1.0

Defines the structure, format, and verification model of AuditVisa evidence outputs.

同一结果，必须可复现、可验证、可传递。
---

1. Purpose

This specification defines the structure and composition of AuditVisa evidence outputs, ensuring that each generated result is verifiable, consistent, and transferable across systems.

---

2. Evidence Components

An AuditVisa evidence package consists of the following elements:

  • HTML report
  • ZIP package
  • JSON metadata
  • Run hash
  • Optional signature
---

3. File Structure

auditvisa-{id}.zip
├── report.html
├── metadata.json
├── runhash.txt
└── signature.sig (optional)
---

4. Metadata Format

{
  "event_id": "evt_xxx",
  "ref": "av_evt_xxx",
  "timestamp": "2026-03-01T10:00:00Z",
  "target": "https://example.com",
  "status": "completed",
  "run_hash": "abc123...",
  "generator": "AuditVisa v0.2"
}
---

5. Run Hash

Run hash is a deterministic hash generated from the evidence content.

It should include:

  • HTML content
  • JSON metadata
  • Normalized input parameters
---

5.1 Purpose

  • Detect tampering
  • Ensure consistency
  • Enable verification
---

6. Verification Model

Verification should follow these steps:

  1. Extract ZIP
  2. Recalculate run hash
  3. Compare with stored runhash.txt
  4. Verify signature (if present)
---

7. Signature (Optional)

AuditVisa may include Ed25519 signature for enhanced trust. 

signature = Sign(run_hash, private_key)
---

8. Integrity Rules

  • No partial files allowed
  • All components must match run hash
  • Any modification invalidates the evidence
---

9. Reproducibility

Given the same input, AuditVisa should generate identical evidence output.

---

10. Delivery

Evidence may be delivered via:

  • Direct download URL
  • API response
  • Archive distribution
---

11. Positioning

Evidence defines what was produced. Idempotency defines how it was produced.
---
