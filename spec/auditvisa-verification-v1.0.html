AuditVisa Verification Specification v1.0

Defines how AuditVisa evidence is verified across local, remote, and third-party contexts.

验证不是“相信”，而是“重算并比对”。
---

1. Purpose

This specification defines the verification process for AuditVisa evidence, ensuring that any party can independently validate the integrity and authenticity of a result.

---

2. Verification Types

  • Local verification (offline)
  • Remote verification (web-based)
  • Third-party verification (multi-node)
---

3. Input Requirements

Verification requires:

  • ZIP evidence package
  • metadata.json
  • runhash.txt
  • signature (optional)
---

4. Local Verification

  1. Extract ZIP
  2. Read metadata.json
  3. Recompute run hash
  4. Compare with runhash.txt

If all match → valid

---

5. Run Hash Validation

Run hash must be recalculated using the same deterministic method:

hash = SHA256(
  normalized(metadata.json) +
  normalized(report.html)
)
---

6. Signature Verification

If signature is present:

verify(signature, run_hash, public_key)

Public key should be retrieved from: 

https://aiorigins.org/.well-known/auditvisa.pub
---

7. Remote Verification

A verification page may allow users to upload a ZIP file and automatically perform:

  • Run hash recalculation
  • Signature validation
  • Result display
---

8. Verification Output

Verification result must clearly indicate:

  • Valid / Invalid
  • Run hash match status
  • Signature status
  • Timestamp
---

9. Failure Conditions

  • Missing files
  • Hash mismatch
  • Signature invalid
  • Corrupted package
---

10. Trust Model

Verification does not require trusting the issuer. It only requires reproducibility and cryptographic consistency.
---

11. Multi-Node Verification (Future)

Future versions may support:

  • Multiple signatures
  • Independent node verification
  • Consensus-based validation
---

12. Positioning

Idempotency controls execution. Evidence defines output. Verification confirms truth.
---
